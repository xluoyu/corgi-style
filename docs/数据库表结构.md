# ClosetAI 数据库表结构

> **项目**：Corgi Style / 衣橱管家
> **版本**：V1.0
> **日期**：2026-03-23
> **状态**：正式版

---

## 一、数据库概览

本项目使用 **PostgreSQL** 数据库，通过 SQLAlchemy ORM 进行数据访问。

### 1.1 表清单

| 序号 | 表名 | 中文名 | 说明 |
|:---:|------|--------|------|
| 1 | `users` | 用户表 | 存储用户基本信息，通过浏览器指纹识别 |
| 2 | `user_profiles` | 用户画像表 | 存储用户的个性化偏好数据 |
| 3 | `clothing_items` | 衣物表 | 核心资产表，存储用户衣橱中的每件衣物 |
| 4 | `outfit_histories` | 穿搭历史表 | 记录用户采纳的穿搭方案快照 |
| 5 | `outfit_feedback` | 穿搭反馈表 | 存储用户对穿搭的评分和反馈 |
| 6 | `outfit_cache` | 穿搭缓存表 | 今日主打推荐的图片缓存 |

### 1.2 ER 关系图


## 二、表结构详解

### 2.1 users - 用户表

**用途**：存储用户基本信息，通过浏览器指纹识别，无需传统登录。

| 字段名 | 类型 | 约束 | 中文说明 |
|--------|------|------|----------|
| `id` | UUID | PK, DEFAULT gen_random_uuid() | 主键，用户唯一标识 |
| `device_fingerprint` | VARCHAR(255) | UNIQUE, NOT NULL, INDEX | 浏览器指纹，用于识别用户设备 |
| `last_active_at` | TIMESTAMP | DEFAULT NOW() | 最后活跃时间 |
| `created_at` | TIMESTAMP | DEFAULT NOW() | 用户创建时间 |
| `updated_at` | TIMESTAMP | DEFAULT NOW() | 最后更新时间 |

**索引**：
- `idx_users_fingerprint` - 浏览器指纹索引，加速用户查找

---

### 2.2 user_profiles - 用户画像表

**用途**：存储用户的个性化偏好数据，如风格、季节、场合等偏好设置。

| 字段名 | 类型 | 约束 | 中文说明 |
|--------|------|------|----------|
| `id` | UUID | PK | 主键 |
| `user_id` | UUID | FK → users.id, UNIQUE, NOT NULL | 关联的用户ID |
| `gender` | VARCHAR(20) | NULL | 性别：male/female/other |
| `style_preferences` | TEXT | NULL | 风格偏好列表，JSON格式存储 |
| `season_preference` | TEXT | NULL | 季节偏好列表，JSON格式存储 |
| `default_occasion` | VARCHAR(50) | DEFAULT 'casual' | 默认场合 |
| `created_at` | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| `updated_at` | TIMESTAMP | DEFAULT NOW() | 更新时间 |

**style_preferences 示例**：
```json
["minimalist", "business", "casual", "sporty"]
```

**season_preference 示例**：
```json
["spring", "autumn"]
```

**场合枚举值**：
- `casual` - 日常休闲
- `work` - 工作通勤
- `formal` - 正式场合
- `sport` - 运动
- `date` - 约会
- `party` - 聚会

---

### 2.3 clothing_items - 衣物表

**用途**：核心资产表，存储用户衣橱中的每件衣物，包括原始图片、卡通图片、属性标签等。

| 字段名 | 类型 | 约束 | 中文说明 |
|--------|------|------|----------|
| `id` | UUID | PK | 主键 |
| `user_id` | UUID | FK → users.id, NOT NULL, INDEX | 关联的用户ID |
| `category` | VARCHAR(50) | NOT NULL | 一级分类 |
| `sub_category` | VARCHAR(100) | NULL | 二级分类 |
| `original_image_url` | VARCHAR(500) | NOT NULL | 原始图片URL |
| `cartoon_image_url` | VARCHAR(500) | NULL | 卡通化图片URL |
| `color` | VARCHAR(32) | NULL | 主要颜色 |
| `material` | VARCHAR(64) | NULL | 材质/面料 |
| `temperature_range` | VARCHAR(50) | NULL | 适合温度范围 |
| `tags` | TEXT | NULL | 属性标签，JSON格式 |
| `wear_count` | INTEGER | DEFAULT 0 | 穿着次数 |
| `last_worn_at` | TIMESTAMP | NULL | 最后穿着时间 |
| `is_deleted` | BOOLEAN | DEFAULT FALSE | 软删除标记 |
| `deleted_at` | TIMESTAMP | NULL | 删除时间（3天后彻底清理） |
| `analysis_completed` | INTEGER | DEFAULT 0 | AI分析是否完成：0-未完成，1-已完成 |
| `generated_completed` | INTEGER | DEFAULT 0 | 卡通图生成是否完成：0-未完成，1-已完成 |
| `created_at` | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| `updated_at` | TIMESTAMP | DEFAULT NOW() | 更新时间 |

**category 一级分类枚举值**：
| 值 | 中文说明 |
|----|----------|
| `top` | 上装（T恤、衬衫、卫衣等） |
| `pants` | 下装（牛仔裤、休闲裤等） |
| `outer` | 外套（夹克、大衣等） |
| `inner` | 内搭（背心、打底等） |
| `accessory` | 配饰（帽子、围巾、珠宝等） |

**sub_category 二级分类示例**：
| category | sub_category |
|----------|--------------|
| top | t-shirt, shirt, blouse, sweater, hoodie, vest |
| pants | jeans, pants, shorts, skirt |
| outer | jacket, coat, cardigan |
| inner | tank_top, undershirt |
| accessory | bag, hat, scarf, jewelry, belt |

**temperature_range 温度范围枚举值**：
| 值 | 中文说明 |
|----|----------|
| `summer` | 夏季（25°C以上） |
| `spring_autumn` | 春秋（10-25°C） |
| `winter` | 冬季（10°C以下） |
| `all_season` | 四季通用 |

**tags JSONB 结构示例**：
```json
{
  "colors": ["white", "blue"],
  "material": "cotton",
  "styles": ["casual", "business"],
  "occasions": ["work", "daily"],
  "seasons": ["spring", "autumn"],
  "thickness": 3
}
```

**索引**：
- `idx_clothing_user_id` - 用户ID索引
- `idx_clothing_category` - 分类索引
- `idx_clothing_deleted` - 软删除过滤索引（仅显示未删除）
- `idx_clothing_tags` - GIN索引，用于JSONB标签查询

### 2.4 outfit_histories - 穿搭历史表

**用途**：记录用户采纳的穿搭方案快照，包括场合、天气快照、选用的衣物等。穿搭的衣物数据以 JSONB 格式存储在 `outfit_snapshot` 字段中。

| 字段名 | 类型 | 约束 | 中文说明 |
|--------|------|------|----------|
| `id` | UUID | PK | 主键 |
| `user_id` | UUID | FK → users.id, NOT NULL, INDEX | 关联的用户ID |
| `occasion` | VARCHAR(50) | NOT NULL | 场合 |
| `outfit_name` | VARCHAR(100) | NULL | 穿搭名称/备注，如"周一通勤搭配" |
| `outfit_snapshot` | TEXT | NULL | 穿搭快照，JSON格式存储完整衣物数据 |
| `weather_snapshot` | TEXT | NULL | 当天天气快照 |
| `create_time` | TIMESTAMP | DEFAULT NOW() | 创建时间 |

**occasion 场合枚举值**：
- `casual` - 日常休闲
- `work` - 工作通勤
- `formal` - 正式场合
- `sport` - 运动
- `date` - 约会
- `party` - 聚会

**outfit_snapshot JSONB 结构示例**：
```json
{
  "top": {
    "id": "uuid",
    "category": "top",
    "sub_category": "shirt",
    "cartoon_image_url": "https://...",
    "tags": {...}
  },
  "bottom": {
    "id": "uuid",
    "category": "bottom",
    "sub_category": "jeans",
    "cartoon_image_url": "https://...",
    "tags": {...}
  },
  "shoes": {
    "id": "uuid",
    "category": "shoes",
    "sub_category": "sneakers",
    "cartoon_image_url": "https://...",
    "tags": {...}
  },
  "outerwear": null,
  "accessory": null
}
```

**weather_snapshot JSONB 结构示例**：
```json
{
  "city": "上海",
  "temperature": 24,
  "condition": "cloudy",
  "condition_text": "多云",
  "humidity": 65,
  "wind_level": 2,
  "rain_probability": 20

```

**索引**：
- `idx_outfit_user_id` - 用户ID索引
- `idx_outfit_created_at` - 创建时间索引
- `idx_outfit_occasion` - 场合索引

---

### 2.5 outfit_feedback - 穿搭反馈表

**用途**：存储用户对穿搭的评分和文字反馈，用于后续推荐优化。

| 字段名 | 类型 | 约束 | 中文说明 |
|--------|------|------|----------|
| `id` | UUID | PK | 主键 |
| `user_id` | UUID | FK → users.id, NOT NULL, INDEX | 关联的用户ID |
| `outfit_id` | UUID | FK → outfit_histories.id, NOT NULL, INDEX | 关联的穿搭记录ID |
| `rating` | INTEGER | NOT NULL | 评分（1-5分） |
| `feedback_text` | TEXT | NULL | 文字反馈 |
| `create_time` | TIMESTAMP | DEFAULT NOW() | 创建时间 |

---

### 2.6 outfit_cache - 穿搭缓存表

**用途**：今日主打推荐的图片缓存，使用 Pexels API 搜索网络时尚穿搭图片，避免重复请求。

| 字段名 | 类型 | 约束 | 中文说明 |
|--------|------|------|----------|
| `id` | INTEGER | PK, AUTOINCREMENT | 主键 |
| `city` | VARCHAR(64) | NOT NULL, INDEX | 城市名 |
| `temperature` | FLOAT | NOT NULL | 温度值（摄氏度） |
| `temperature_range` | VARCHAR(32) | NOT NULL, INDEX | 温度范围 |
| `scene` | VARCHAR(32) | NOT NULL, INDEX | 场景/场合 |
| `image_url` | TEXT | NOT NULL | 穿搭图片URL |
| `description` | TEXT | NULL | 穿搭描述文本 |
| `outfit_items` | JSONB | NULL | 搭配单品列表 |
| `create_time` | TIMESTAMP | DEFAULT NOW() | 创建时间 |

**temperature_range 温度范围**：
- `summer` - 夏季（≥25°C）
- `spring_autumn` - 春秋（10-25°C）
- `winter` - 冬季（<10°C）

**scene 场景枚举值**：
- `daily` - 日常
- `work` - 工作
- `sport` - 运动
- `date` - 约会
- `party` - 聚会

**outfit_items JSONB 结构示例**：
```json
[
  {"slot": "上装", "description": "白色纯棉T恤", "color": "白色"},
  {"slot": "下装", "description": "浅蓝色九分牛仔裤", "color": "浅蓝色"},
  {"slot": "鞋履", "description": "白色低帮帆布鞋", "color": "白色"}
]
```

**缓存查找键**：(city + temperature_range + scene)

## 四、触发器

### 4.1 自动更新 updated_at 触发器

为 `users`、`user_profiles`、`clothing_items` 表创建了 `BEFORE UPDATE` 触发器，自动更新 `updated_at` 字段。

| 触发器名 | 表 | 事件 | 动作 |
|----------|-----|------|------|
| update_users_updated_at | users | BEFORE UPDATE | 更新 updated_at |
| update_profiles_updated_at | user_profiles | BEFORE UPDATE | 更新 updated_at |
| update_clothing_updated_at | clothing_items | BEFORE UPDATE | 更新 updated_at |

## 五、安全策略

### 5.1 Row Level Security (RLS)

启用 RLS 确保用户只能访问自己的数据，防止数据泄露。

**启用 RLS 的表**：
- user_profiles
- clothing_items
- outfit_histories

**策略说明**：
- 用户只能访问 `user_id` 等于自己ID的记录
- 所有数据操作（SELECT/INSERT/UPDATE/DELETE）都受策略限制
- 即使表所有者执行查询也会受 RLS 限制（FORCE ROW LEVEL SECURITY）

### 5.2 用户上下文设置

由于使用浏览器指纹识别而非传统认证，通过应用层设置用户上下文：

```sql
-- 设置当前用户上下文
SELECT set_user_id('user_uuid_here');

-- 获取当前用户ID
SELECT current_user_id();
```

## 七、软删除机制

### 7.1 衣物软删除规则

1. 用户删除衣物时，设置 `is_deleted = TRUE`，`deleted_at = NOW()`
2. 删除后 3 天内可在回收站恢复
3. 3 天后由后台任务彻底删除

### 7.2 清理函数

```sql
-- 清理超过3天的已删除衣物
SELECT cleanup_deleted_clothes();
```
