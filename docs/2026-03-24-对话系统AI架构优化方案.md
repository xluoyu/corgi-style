## 一、问题诊断

### 1.1 当前架构问题

当前对话系统采用"LLM 意图识别 + 硬编码规则兜底"的混合模式，存在以下问题：

#### 问题一：硬编码 Fallback 导致智能感缺失

```python
# response.py 中的硬编码示例
def _is_greeting(message: str) -> bool:
    greetings = {"你好", "您好", "嗨", "hi", "hello", ...}
    return message in greetings  # 无法识别变体

def _is_likely_city(text: str) -> bool:
    common_cities = {"北京", "上海", "广州", ...}  # 维护成本高
    return text in common_cities
```

**影响**：
- 用户说"帝都"无法识别为北京
- 用户说"魔都"无法识别为上海
- 新增城市需要手动维护列表

#### 问题二：多轮对话上下文断裂

```
用户：帮我推荐一套穿搭
AI：好的！想去哪里（城市）呢？
用户：北京
AI：抱歉，这句话我还不太理解 😅  # 上下文丢失
```

**根因分析**（经代码审查修正）：
1. ~~第一轮追问时未保存 context~~ → 实际已有 `DialogueSessionManager` 进行上下文管理
2. **真正原因**：`response.py` 中的 `_handle_unknown()` 未正确读取 session 中的上下文信息
3. 简短回复（如"北京"）被 `intent_node` 识别为 `unknown`，进入 fallback 后无法关联上一轮追问

#### 问题三：LLM 调用分散，成本和延迟高

```
当前流程：
  intent_node (LLM) → planning_node (LLM) → response_node (模板)

延迟：~1.5s
成本：~$0.01/轮
```

#### 问题四：原方案设计存在缺陷（v2.2 修订补充）

经代码审查和方案评审，发现原 v2.1 方案存在以下设计问题：

| 问题 | 描述 | 影响 |
|------|------|------|
| **快速路径误判** | 方案中 `_is_quick_confirmation()` 会将 2 字城市名（如"北京"、"上海"）误判为确认语 | 城市信息丢失 |
| **响应缓存不可行** | 方案使用 MD5 hash 做缓存 key，要求消息完全匹配才能命中 | 实际对话变化大，缓存命中率极低 |
| **FallbackManager 过度设计** | 方案新增全局状态管理器，但 `intent_node` 已有异常捕获机制 | 重复造轮子，增加复杂度 |
| **AI 分类增加成本** | 新增独立的 `_ai_classify_input()` 调用，1 次对话最多 3 次 LLM 调用 | 成本不降反升 |

**评审结论**：原方案大方向正确，但具体实现细节存在设计问题，需按 v2.2 修订版执行。

### 1.2 架构评估

| 维度 | 当前状态 | 评分 | 说明 |
|------|----------|------|------|
| 智能度 | 硬编码 fallback 降低体验 | ⭐⭐ | 城市别名、场景变体无法识别 |
| 可维护性 | 规则分散，难以扩展 | ⭐⭐ | 城市列表、问候语需手动维护 |
| 成本 | 2 次 LLM 调用/轮 | ⭐⭐⭐ | 可接受，但有优化空间 |
| 延迟 | 1.5-2s | ⭐⭐⭐ | 用户可接受，但可优化 |
| 可靠性 | 规则兜底保证稳定性 | ⭐⭐⭐⭐ | 硬编码规则提供稳定降级 |

### 1.3 现有架构优势（需保留）

经代码审查，发现以下已有优势应在优化中保留：

| 模块 | 文件 | 说明 |
|------|------|------|
| Session 管理 | `dialogue_session.py` | 完善的 PostgreSQL 持久化 + 内存缓存 |
| LangGraph 工作流 | `workflow.py` | 清晰的节点编排，易于调试 |
| LLM Provider 缓存 | `llm_providers.py` | `@lru_cache` 避免重复初始化 |
| 规则兜底 | `response.py` | 硬编码规则提供稳定降级 |

## 二、优化方案：渐进式 AI 增强

### 2.1 核心原则

```
┌─────────────────────────────────────────────────────────────┐
│                    设计原则（v2.0 修订）                      │
├─────────────────────────────────────────────────────────────┤
│  1. AI 增强：复杂场景用 LLM，简单场景走快速路径                 │
│  2. 规则保留：硬编码规则作为降级方案，不删除                     │

| 城市识别 | 硬编码城市列表 | LLM 语义理解 + 规则兜底 | 智能度 ↑ | 低 |
| 问候检测 | 关键词匹配 | 快速路径 + LLM 判断 | 准确率 ↑ | 低 |

| 降级机制 | 隐式 | 显式降级策略 | 可靠性 ↑ | 无 |

def _normalize_city(text: str) -> Optional[str]:
    """
    城市名标准化：支持别名和大小写
    
    Args:
        text: 用户输入的城市名
        
    Returns:
        标准城市名（如"北京"）或 None
    """

    # 先检查别名
    if text in CITY_ALIASES:
        return CITY_ALIASES[text]
    # 再检查常见城市
    if text in COMMON_CITIES:
        return text
    return None


def _is_quick_confirmation(message: str) -> bool:
    """判断是否是快速确认语（修订：排除城市名）"""
    msg = message.strip()
    # 排除可能是城市名的输入
    if _normalize_city(msg):
        return False
    quick_confirm = {"好", "行", "嗯", "OK", "ok", "是", "对", "嗯嗯", "好的", "可以"}
    return msg.lower() in quick_confirm


def _is_quick_greeting(message: str) -> bool:
    """判断是否是常见问候语（快速匹配）"""
    quick_greetings = {"你好", "您好", "嗨", "hi", "hello", "在吗", "在不在"}
    msg = message.strip().lower()
    return msg in quick_greetings

def _build_missing_question(state: GraphState, intent_str: str) -> Optional[Tuple[str, Dict]]:
    """检测缺失的关键信息，如果缺少则主动提问用户"""
    if intent_str == "generate_outfit":
        missing = []
        if not state.get("target_city"):
            missing.append("要去哪里（城市）呢")
        if not state.get("target_scene"):
            missing.append("是什么场合（比如上班、约会、运动）")
        if not state.get("user_clothes"):
            missing.append("让我看看您的衣柜里有什么衣服～")

        if missing:
            question = "好的！想帮您搭配得更好，" + missing[0]
            if len(missing) > 1:
                question += "，" + "，".join(missing[1:])
            question += "？"
            return question, {"type": "ask_info", "missing": missing}
    return None


async def response_node(state: GraphState) -> GraphState:
    """
    响应生成节点 - v2.0 增强版
    
    流程：
    1. 快速路径判断
    2. 缓存查询
    3. AI 分类（带降级）
    4. 响应生成
    """
    intent_str = state.get("intent_str") or state.get("intent") or ""
    if hasattr(intent_str, "value"):
        intent_str = intent_str.value
    else:
        intent_str = str(intent_str) if intent_str else ""

    user_message = state.get("messages", [{}])[-1].get("content", "") if state.get("messages") else ""
    
    # 构建上下文（从 session 中读取）
    context = _build_context_from_state(state)

    # 1. 检查缺失信息，主动提问
    missing_q = _build_missing_question(state, intent_str)
    if missing_q:
        response_content, data = missing_q
        # 记录追问，用于下一轮上下文
        state["last_question"] = missing_q[0]
    
    # 2. 快速路径判断
    elif not missing_q:
        quick_response = _get_quick_path_response(user_message, context)
        if quick_response:
            response_content, data = quick_response
        
        # 3. 缓存查询
        else:
            cached = _response_cache.get(user_message, context)
            if cached:
                response_content, data = cached
            
            # 4. 根据意图处理
            else:
                if intent_str == "query_wardrobe":
                    response_content, data = _handle_wardrobe_query(state)
                elif intent_str == "generate_outfit":
                    response_content, data = _handle_generate_outfit(state)
                elif intent_str == "give_feedback":
                    response_content, data = _handle_feedback(state)
                elif intent_str == "get_advice":
                    response_content, data = _handle_get_advice(state)
                else:
                    # unknown 意图：使用增强的 AI 分类
                    response_content, data = await _handle_unknown_enhanced(state, context)

    messages = state.get("messages", [])
    messages.append({"role": "assistant", "content": response_content})
    state["messages"] = messages

    state["response_data"] = data
    state["should_end"] = True

    return state

#### 3.2.4 多轮对话信息收集机制（关键改进）

**问题场景**：
```
用户：帮我推荐明天的穿搭
AI：你明天要去哪里？参加什么活动？
用户：明天去北京
AI：参加什么场景呢？
用户：我要去开会
AI：[生成穿搭方案]
```

**核心问题**：当前方案缺少"追问状态"管理，导致：
1. 一次性问多个问题，用户体验差
2. 用户回答追问时，intent_node 无法感知上下文
3. 简短回复（如"北京"）可能被识别为 unknown

**解决方案**：引入"追问状态机"

```
┌─────────────────────────────────────────────────────────────────────┐
│                    追问状态机                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  状态流转：                                                          │
│                                                                      │
│  [初始状态]                                                          │
│      │                                                               │
│      ↓ 用户请求穿搭推荐                                               │
│  [检查缺失信息]                                                       │
│      │                                                               │
│      ├── 缺少城市 → asking_for = "city" → 追问城市                    │
│      │                                                               │
│      ├── 缺少场景 → asking_for = "scene" → 追问场景                   │
│      │                                                               │
│      └── 信息完整 → asking_for = None → 生成穿搭                      │
│                                                                      │
│  用户回答追问时：                                                      │
│      │                                                               │
│      ├── asking_for = "city" → 提取城市 → 继续检查缺失                 │
│      │                                                               │
│      └── asking_for = "scene" → 提取场景 → 继续检查缺失                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

def _build_missing_question(state: GraphState, intent_str: str) -> Optional[Tuple[str, Dict[str, Any]]]:
    """
    检测缺失的关键信息，逐个追问
    
    Args:
        state: 当前对话状态
        intent_str: 意图类型
        
    Returns:
        (question_text, data_dict) 如果需要追问，否则 None

    if intent_str != "generate_outfit":

        question = "好的！您要去哪个城市呢？"
        return question, {
            "type": "ask_info",
            "missing": ["city"],
            "asking_for": "city",
            "pending_intent": "generate_outfit"
        }
    
    if not state.get("target_scene"):
        city = state.get("target_city", "")
        question = f"好的，{city}不错～您是去什么场合呢？（比如上班、约会、运动）"
        return question, {
            "type": "ask_info",
            "missing": ["scene"],
            "asking_for": "scene",
            "pending_intent": "generate_outfit"
        }
    
    if not state.get("user_clothes"):
        question = "让我看看您的衣柜里有什么衣服～"
        return question, {
            "type": "ask_info",
            "missing": ["wardrobe"],
            "asking_for": "wardrobe",
            "pending_intent": "generate_outfit"
        }
    
    return None

【重要】请结合上下文理解用户意图：
- 如果正在追问城市（asking_for=city），用户回答"北京"→ intent=generate_outfit, city=北京
- 如果正在追问场景（asking_for=scene），用户回答"开会"→ intent=generate_outfit, scene=work

- 已知城市：{known_city}
- 已知场景：{known_scene}
- 已知日期：{known_date}
- 正在追问：{asking_for}

请分析用户输入，提取以下信息：
1. intent: 意图类型（只能选一个）

   - get_advice: 用户想获取搭配建议
   - give_feedback: 用户对已有穿搭提反馈
   - unknown: 无法判断

   - date: 目标日期（如"后天"转为具体日期 2026-03-25）
   - city: 城市名称
   - scene: 场景（只能是以下值之一：daily, work, sport, date, party, casual, formal）
   - temperature: 温度
   - style: 风格偏好

注意：

- "明天去杭州出差穿什么" → date=明天, city=杭州, scene=work
- "开会" → 如果 asking_for=scene，则 scene=work
- "帝都" → city=北京
- "魔都" → city=上海

输出JSON格式，只输出JSON，不要有其他文字。"""


def create_intent_prompt(message: str, context: Dict[str, Any] = None) -> str:
    """
    构建意图识别 prompt（增强版）
    
    Args:
        message: 用户输入
        context: 上下文信息，包含 asking_for, pending_intent 等
        
    Returns:
        完整的 prompt 字符串
    """
    context = context or {}
    
    # 提取上下文信息
    known_city = context.get("target_city", "未知")
    known_scene = context.get("target_scene", "未知")
    known_date = context.get("target_date", "未知")
    asking_for = context.get("asking_for", "无")
    pending_intent = context.get("pending_intent", "无")
    
    return INTENT_SYSTEM_PROMPT.format(
        known_city=known_city,
        known_scene=known_scene,
        known_date=known_date,
        asking_for=asking_for,
        pending_intent=pending_intent
    ) + f"\n\n用户输入：{message}"


async def intent_node(state: GraphState) -> GraphState:
    """
    意图识别节点（增强版）
    
    使用 LLM 分析用户消息，提取意图和实体
    支持追问状态感知
    """
    user_message = state.get("messages", [])[-1]["content"] if state.get("messages") else ""
    
    # 获取上下文（包含追问状态）
    context = state.get("context", {})
    
    # 构建 prompt（包含追问状态）
    prompt = create_intent_prompt(user_message, context)
    
    try:
        llm = get_cached_provider()
        chat_model = llm.chat_model
        
        logger.info(f"[LLM] intent识别开始 | message={user_message[:50]} | asking_for={context.get('asking_for')}")
        response = await chat_model.ainvoke([HumanMessage(content=prompt)])
        logger.info(f"[LLM] intent识别完成 | response={response.content[:100]}")
        
        result = parse_intent_response(response.content)
    except Exception as e:
        result = {
            "intent": Intent.UNKNOWN,
            "entities": {},
            "confidence": 0.0
        }
        state["error"] = f"意图识别失败: {str(e)}"
        logger.error(f"[LLM] intent识别失败 | error={e}")
    
    # 更新状态
    state["intent"] = result["intent"]
    state["intent_str"] = getattr(result["intent"], "value", None) if result["intent"] else None
    state["entities"] = result["entities"]
    state["intent_confidence"] = result["confidence"]
    
    # 从 entities 中提取关键信息到顶层
    entities = result["entities"]
    if "date" in entities:
        state["target_date"] = entities["date"]
    if "city" in entities:
        state["target_city"] = entities["city"]
    if "scene" in entities:
        state["target_scene"] = entities["scene"]
    
    # 关键：如果用户正在回答追问，保持 pending_intent
    asking_for = context.get("asking_for")
    if asking_for and result["intent"] != Intent.UNKNOWN:
        # 用户回答了追问，继承之前的意图
        state["pending_intent"] = context.get("pending_intent")
        state["intent_str"] = context.get("pending_intent")
    
    return state

async def response_node(state: GraphState) -> GraphState:
    """
    响应生成节点（增强版）
    
    支持追问状态管理和多轮信息收集
    """
    intent_str = state.get("intent_str")
    user_message = state.get("messages", [{}])[-1].get("content", "")
    context = state.get("context", {})
    
    # 获取追问状态
    asking_for = context.get("asking_for")
    pending_intent = context.get("pending_intent")
    
    # ========== 关键改进：处理追问回答 ==========
    if asking_for and pending_intent:
        # 用户正在回答追问
        logger.info(f"[Response] 处理追问回答 | asking_for={asking_for} | pending_intent={pending_intent}")
        
        # 使用 AI 分类提取用户回答的内容
        classification = await _ai_classify_input(user_message, context)
        
        if classification["type"] == asking_for:
            # 成功提取回答
            if asking_for == "city":
                state["target_city"] = classification["value"]
                state["context"]["target_city"] = classification["value"]
            elif asking_for == "scene":
                state["target_scene"] = classification["value"]
                state["context"]["target_scene"] = classification["value"]
            
            # 清除追问状态
            state["asking_for"] = None
            state["context"]["asking_for"] = None
            
            # 继续检查是否还有缺失信息
            missing_question = _build_missing_question(state, pending_intent)
            if missing_question:
                question, data = missing_question
                state["response"] = question
                state["response_data"] = data
                state["asking_for"] = data.get("asking_for")
                state["context"]["asking_for"] = data.get("asking_for")
            else:
                # 信息完整，生成穿搭
                state["intent_str"] = pending_intent
                return await _handle_generate_outfit(state)
        else:
            # 用户回答与追问无关，按正常流程处理
            logger.info(f"[Response] 回答与追问无关 | expected={asking_for} | got={classification['type']}")
    
    # ========== 正常流程 ==========
    # 检查是否需要追问
    missing_question = _build_missing_question(state, intent_str)
    if missing_question:
        question, data = missing_question
        state["response"] = question
        state["response_data"] = data
        state["asking_for"] = data.get("asking_for")
        state["pending_intent"] = intent_str
        # 同步到 context
        state["context"]["asking_for"] = data.get("asking_for")
        state["context"]["pending_intent"] = intent_str
        return state
    
    # 正常处理各种意图
    if intent_str == "generate_outfit":
        return await _handle_generate_outfit(state)
    elif intent_str == "query_wardrobe":
        return _handle_wardrobe_query(state)
    elif intent_str == "unknown":
        return _handle_unknown(state)
    
    return state

"""条件分支函数 - v2.0 增强版"""
from typing import Literal, Optional
from app.agent.graph.state import GraphState, Intent


def route_by_intent(state: GraphState) -> Literal["wardrobe_query", "generate_outfit", "feedback", "response"]:
    """根据意图类型路由"""
    intent_str = _get_intent_str(state)

    if intent_str == "query_wardrobe":
        return "wardrobe_query"
    elif intent_str == "generate_outfit":
        return "generate_outfit"
    elif intent_str == "give_feedback":
        return "feedback"
    else:
        # unknown / greeting / confirmation 等直接响应
        return "response"


def route_after_intent(state: GraphState) -> Literal["planning", "response"]:
    """
    意图识别后的路由（新增）
    
    简单意图直接响应，跳过 planning 节点
    """
    intent_str = _get_intent_str(state)
    
    # 简单意图直接响应
    if intent_str in ["query_wardrobe", "unknown"]:
        return "response"
    
    # 复杂意图走 planning
    return "planning"


def _get_intent_str(state: GraphState) -> Optional[str]:
    """安全获取 intent 字符串值"""
    raw = state.get("intent_str") or state.get("intent")
    if raw is None:
        return None
    if hasattr(raw, "value"):
        return raw.value
    if isinstance(raw, str):
        return raw
    return str(raw)
```

"""天气节点 - 支持并行"""
import asyncio
from app.agent.graph.state import GraphState
from app.agent.services.weather_service import WeatherService


async def weather_node(state: GraphState) -> GraphState:
    """
    天气节点
    
    如果已有温度信息，跳过 API 调用
    """
    # 如果已有温度，直接返回
    if state.get("target_temperature"):
        return state
    
    city = state.get("target_city")
    date = state.get("target_date")
    
    if not city:
        return state
    
    # 调用天气服务
    weather_service = WeatherService()
    try:
        weather_data = await weather_service.get_weather(city, date)
        state["target_temperature"] = weather_data.get("temperature")
        state["weather_condition"] = weather_data.get("condition")
    except Exception as e:
        logger.warning(f"天气查询失败: {e}")
        # 降级：使用默认温度
        state["target_temperature"] = 20.0
    
    return state

## 十、总结

本方案采用 **渐进式 AI 增强** 策略，核心思路是：

1. **保留优势**：不删除现有规则，保留作为降级方案
2. **快速路径**：简单场景（问候、确认）直接响应，不走 LLM
3. **城市别名**：轻量级映射表支持"帝都"→"北京"等别名识别
4. **规则兜底**：简化版规则分类器替代独立的 AI 分类调用
5. **追问状态**：完善的 asking_for 机制保证多轮对话上下文

**v2.2 修订说明**：
- 删除了不可行的响应缓存设计（KV 缓存不适用于对话系统）
- 删除了过度设计的 FallbackManager（复用 intent_node 现有异常处理）
- 删除了新增 AI 分类调用的方案（成本不降反升）
- 新增轻量级城市别名映射方案

预期收益：
- **用户体验**：25% 请求（问候、确认、城市名）延迟显著降低
- **成本控制**：LLM 调用减少约 18%
- **可靠性**：复用现有降级机制，保证服务稳定性
