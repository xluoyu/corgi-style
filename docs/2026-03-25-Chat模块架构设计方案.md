## 一、背景与目标

### 1.1 项目背景

当前 Chat 模块需要升级为一个智能 Agent 系统，支持多种技能，包括天气查询、图片识别、衣柜管理、穿搭推荐等功能。

### 1.2 设计目标

1. **多技能支持**：支持天气查询、图片识别、衣柜管理、穿搭推荐等多种技能
2. **多模态输入**：同时支持文本和图片输入
3. **智能路由**：基于意图自动路由到对应技能，减少硬编码
4. **可扩展性**：易于添加新技能，模块化设计
5. **复用现有代码**：最大化利用已有的 LangGraph、天气服务、图片分析等模块

### 1.3 核心功能需求

| 技能   | 功能描述                       | 触发条件         |
| ---- | -------------------------- | ------------ |
| 天气查询 | 查询指定城市和日期的天气，附带穿衣指南        | "北京明天天气怎么样"  |
| 图片识别 | 识别图片中衣物的材质、样式、颜色等信息，提供搭配建议 | 用户上传图片       |
| 衣柜管理 | 存储衣物到衣柜、查询衣柜内容、衣柜统计        | "我衣柜里有什么"    |
| 穿搭推荐 | 根据天气、场景从衣柜中推荐穿搭方案          | "明天去北京出差穿什么" |
| 历史查询 | 查询用户的穿搭历史记录                | "我上周穿了什么"    |
| 知识问答 | 基于知识库的穿搭知识问答               | "春天怎么搭配颜色"   |
| 自由对话 | 与 LLM 自由对话                 | 闲聊、打招呼       |

### 2.2 核心设计理念

1. **模块化**：每个技能独立封装，易于维护和扩展
2. **智能路由**：基于意图的自动路由，减少硬编码
3. **多模态**：支持文本和图片混合输入
4. **可扩展**：新增技能只需实现标准接口
5. **复用性**：最大化利用现有代码和基础设施

## 三、技能详细设计

### 3.1 技能一：天气查询

#### 触发条件

- 用户询问天气相关："北京明天天气怎么样"、"后天去上海穿什么"
- 作为穿搭推荐的依赖服务

#### 执行流程

```
用户输入 → 意图识别 → 提取城市+日期 → 调用天气API → 生成穿衣指南 → 返回结果
```

#### 数据结构

```python
class WeatherResponse:
    """天气响应数据结构"""
    city: str              # "北京"
    date: str              # "2026-03-25"
    temperature: float     # 18.5
    weather: str           # "多云"
    humidity: int          # 45
    wind: str              # "北风3级"
    
    # 穿衣指南
    clothing_advice: str   # "建议穿薄外套，早晚温差较大"
    temperature_range: str # "spring_autumn"
```

#### 实现要点

- 复用现有的 `weather_service.py`
- 新增 `ClothingAdviceGenerator` 类，根据天气生成穿衣建议
- 支持未来 3 天天气查询
- 集成高德天气 API

#### 示例对话

```
用户：北京明天天气怎么样？
Agent：明天北京多云，气温 12-18℃，北风 3 级。
      穿衣建议：建议穿薄外套或卫衣，早晚温差较大，可准备一件轻便外套备用。
```

### 3.2 技能二：图片识别

#### 触发条件

- 用户上传图片 + 文本描述（可选）
- 意图识别为 `analyze_image` 或 `save_to_wardrobe`

#### 两种场景

##### 场景 A：纯识别（不存储）

```
用户上传图片 → 图片分析 → 提取衣物属性 → 生成搭配建议 → 返回结果
```

##### 场景 B：识别并存储

```
用户上传图片 + "存到衣柜" → 图片分析 → 生成商品图 → 存储到DB → 返回确认
```

#### 数据结构

```python
class ImageAnalysisResult:
    """图片识别结果"""
    # 基础属性
    name: str              # "蓝色条纹衬衫"
    category: str          # "top"
    color: str             # "蓝色"
    material: str          # "棉"
    style: str             # "休闲"
    
    # 适用场景
    suitable_scenes: List[str]  # ["daily", "work", "date"]
    suitable_weather: List[str] # ["spring_autumn", "summer"]
    temperature_range: str      # "spring_autumn"
    
    # 搭配建议
    matching_suggestions: List[Dict]  # [{"item": "深色牛仔裤", "reason": "..."}]
```

#### 实现要点

- 复用现有的 `ImageAnalyzer` 和 `image_generator.py`
- 新增 `ImageSkillHandler` 统一处理图片相关请求
- 支持 OSS 图片上传和临时 URL 生成
- 使用通义千问 VL 进行图片分析

#### 示例对话

```
用户：[上传图片] 帮我看看这件衣服
Agent：我识别到这是一件蓝色条纹衬衫，材质为棉质，适合春夏季节穿着。
      
      适合场景：日常、通勤、约会
      适合天气：15-25℃
      
      搭配建议：
      1. 深色牛仔裤 + 白色运动鞋（休闲风格）
      2. 卡其色休闲裤 + 乐福鞋（商务休闲）
      3. 黑色西装裤 + 德比鞋（正式场合）
```

### 4.3 技能路由器设计

#### 路由器实现

```python
class SkillRouter:
    """技能路由器"""
    
    def __init__(self):
        self.skills = {
            "weather": WeatherSkill(),
            "image": ImageSkill(),
            "wardrobe": WardrobeSkill(),
            "outfit": OutfitSkill(),
            "history": HistorySkill(),
            "knowledge": KnowledgeSkill(),
            "chat": ChatSkill(),
        }
    
    async def route(self, state: GraphState) -> str:
        """根据意图路由到对应技能"""
        intent = state.get("intent")
        has_image = state.get("image_url") or state.get("image_data")
        
        # 图片优先判断
        if has_image:
            if "存" in state.get("message", "") or "衣柜" in state.get("message", ""):
                return "image:save"
            return "image:analyze"
        
        # 意图路由
        route_map = {
            Intent.QUERY_WEATHER: "weather",
            Intent.ANALYZE_IMAGE: "image:analyze",
            Intent.SAVE_TO_WARDROBE: "image:save",
            Intent.QUERY_WARDROBE: "wardrobe",
            Intent.GENERATE_OUTFIT: "outfit",
            Intent.QUERY_HISTORY: "history",
            Intent.ASK_KNOWLEDGE: "knowledge",
            Intent.CHAT: "chat",
        }
        
        return route_map.get(intent, "chat")
```

#### 技能基类

```python
class BaseSkill(ABC):
    """技能基类"""
    
    @abstractmethod
    async def execute(self, state: GraphState) -> GraphState:
        """执行技能"""
        pass
    
    @abstractmethod
    def can_handle(self, state: GraphState) -> bool:
        """判断是否可以处理"""
        pass
```

CREATE TABLE outfit_history (

## 九、技术选型建议

### 9.1 技术栈对比

| 组件     | 技术选型              | 说明        |
| ------ | ----------------- | --------- |
| LLM    | 通义千问 / GPT-4      | 意图识别、搭配推理 |
| 图片分析   | 通义千问 VL           | 衣物属性识别    |
| 图片生成   | Stable Diffusion  | 商品图、搭配效果图 |
| 向量数据库  | pgvector / Milvus | 知识库 RAG   |
| 天气 API | 高德天气              | 实时天气数据    |
| 存储     | OSS               | 图片存储      |
| 数据库    | PostgreSQL        | 业务数据存储    |
| 框架     | LangGraph         | 工作流编排     |

### 9.2 选型理由

#### LLM 选型

- **通义千问**：国内访问稳定，支持中文，性价比高
- **GPT-4**：推理能力强，适合复杂场景

#### 图片分析选型

- **通义千问 VL**：支持图片理解，中文友好，API 稳定

#### 向量数据库选型

- **pgvector**：与 PostgreSQL 集成，部署简单
- **Milvus**：性能更好，适合大规模场景

## 十一、总结与建议

### 11.1 架构优势

1. **模块化设计**：每个技能独立封装，易于维护和扩展
2. **多模态支持**：同时处理文本和图片输入
3. **智能路由**：基于意图的自动路由，减少硬编码
4. **可扩展性**：新增技能只需实现标准接口
5. **复用现有代码**：最大化利用已有的 LangGraph、天气服务、图片分析等模块

### 11.2 风险与挑战

1. **意图识别准确性**：需要大量测试和优化
2. **图片分析成本**：API 调用成本较高，需要优化
3. **知识库建设**：需要积累大量穿搭知识
4. **性能优化**：多技能串联可能影响响应速度

### 11.3 下一步建议

建议按以下顺序实施：

1. **先实现核心技能**：天气查询、图片识别、穿搭推荐
2. **逐步扩展**：历史查询、知识库
3. **持续优化**：根据用户反馈迭代
